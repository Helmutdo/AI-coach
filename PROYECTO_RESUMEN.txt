# Garmin Trainer — Resumen del Proyecto

## Descripción
Plataforma full-stack de analítica deportiva para atletas de resistencia. Integra datos de Garmin Connect y Strava con un coach de IA (Claude, GPT-4o o Gemini) que analiza el entrenamiento y responde preguntas personalizadas.

---

## Stack Tecnológico

**Backend — FastAPI (Python)**
- FastAPI + Uvicorn (ASGI)
- SQLAlchemy ORM + SQLite (dev) / PostgreSQL (prod)
- `garminconnect` — integración con Garmin Connect vía Garth OAuth
- `httpx` — cliente async para Strava API
- Anthropic / OpenAI / Google AI — multi-provider AI
- Fernet encryption — cifrado de tokens y API keys
- `slowapi` — rate limiting

**Frontend — Next.js 14 (TypeScript)**
- Next.js App Router + React 18
- NextAuth v5 — autenticación con Google OAuth
- Zustand — state management
- Recharts — gráficos
- TailwindCSS

## Modelos de Base de Datos

| Modelo | Campos clave |
|--------|-------------|
| `User` | `google_id`, `email`, `name`, `avatar_url` |
| `UserSettings` | `garmin_token_encrypted`, `strava_*_token_encrypted`, `ai_provider`, `ai_api_key_encrypted` |
| `GarminActivity` | `activity_type`, `duration`, `distance`, `avg_hr`, `training_load`, `aerobic_effect` |
| `StravaActivity` | `sport_type`, `elapsed_time`, `distance`, `avg_watts`, `suffer_score`, `map_polyline` |
| `DailyMetrics` | `resting_hr`, `sleep_duration`, `sleep_score`, `vo2max`, `hrv_status`, `body_battery` |
| `ChatMessage` | `role`, `content`, `context_snapshot` (snapshot de métricas en el momento) |

## Endpoints API (Backend puerto 8000)

```
POST   /auth/garmin/login          # Conectar cuenta Garmin
GET    /auth/garmin/status         # Estado conexión Garmin
POST   /auth/ai/configure          # Configurar provider IA + API key
POST   /users/me                   # Upsert perfil (desde Google OAuth)
DELETE /users/me/data              # Borrar todos los datos sincronizados

POST   /garmin/sync                # Sync actividades y métricas (rate: 5/min)
GET    /garmin/activities          # Listar actividades Garmin
GET    /garmin/daily-metrics       # Métricas diarias (sueño, estrés, HRV)
GET    /garmin/summary             # Resumen semanal/mensual

GET    /strava/connect             # URL OAuth Strava
GET    /strava/callback            # Callback OAuth Strava
POST   /strava/sync                # Sync actividades Strava (60 días)
GET    /strava/status              # Estado + perfil atleta Strava

POST   /coach/chat                 # Mensaje al coach IA
GET    /coach/analysis             # Análisis IA (7/30/90 días)
GET    /coach/history              # Historial de conversación
```

## Flujo de Autenticación
1. Usuario hace login con **Google OAuth** en Next.js (NextAuth)
2. Frontend sincroniza perfil con backend vía `POST /users/me` — el `google_id` actúa como identificador
3. Todas las requests autenticadas envían `X-User-Id` header al backend
4. Garmin y Strava se conectan por separado en Settings, con tokens cifrados en DB

---

## Características del Coach IA
- Multi-provider: Anthropic `claude-sonnet-4`, OpenAI `gpt-4o`, Google `gemini-2.0-flash`
- Contexto: últimos 30-90 días de actividades + métricas diarias inyectadas al prompt
- System prompt: coach experto en deportes de resistencia
- Análisis automático sin chat: endpoint `/coach/analysis`
- Historial persistido por `conversation_id`

---

## Analytics del Dashboard
- **PMC** (Performance Management Chart): CTL, ATL, TSB
- Distribución de zonas de frecuencia cardíaca
- Volumen semanal por deporte
- Training Stress Score (TSS)
- Métricas de sueño y recuperación
- Carga aguda vs crónica
